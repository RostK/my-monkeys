# `schemas/` — vendored Claude Code JSON Schemas

This directory contains **committed, byte-identical copies** of the two upstream Claude Code JSON
Schemas that `npm run validate:manifests` validates the catalog against:

- `claude-code-marketplace.json` — validates `.claude-plugin/marketplace.json`.
- `claude-code-plugin-manifest.json` — validates every `plugins/*/.claude-plugin/plugin.json`.

## Do not hand-edit these files

**`claude-code-marketplace.json` and `claude-code-plugin-manifest.json` must never be edited by
hand — not even to add a comment or an `x-vendored-from`-style provenance key.** They must stay
**byte-identical** to the upstream document they were fetched from. Any change — including
whitespace or an added annotation — makes the committed copy permanently differ from upstream, and
the scheduled drift check (`npm run schemas:check`) will report drift forever even though nothing
upstream ever changed.

Upstream provenance (the source URL, the date it was vendored, and its sha256) is recorded
**outside** the schema bodies, in [`provenance.json`](./provenance.json) — precisely so the schema
copies themselves stay pristine.

## Updating a vendored copy

```bash
npm run schemas:check   # fetch upstream, diff against the committed copy, exit non-zero on drift
npm run schemas:update  # fetch upstream, overwrite the committed copy, restamp provenance.json
```

Both commands are the same script (`scripts/vendor-schemas.mjs`) run in `--check` / `--write` mode.
Never edit `provenance.json` by hand either — it is regenerated by `schemas:update`.

## Why vendor instead of validating against a live URL

Validating against a URL fetched at CI run time would make a third-party, community-run host
(`json.schemastore.org` — not an Anthropic domain) a hard dependency of every pull request that
touches the catalog: its outage would block every merge, and an edit to that document would turn
CI red with no change in this repository. Vendoring moves that third-party risk off the merge path
and onto a scheduled, non-gating drift check instead.
